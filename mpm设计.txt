模块包格式如下：
modulepack.zip
     moudule1
          manifest.yml
          scripts
               main.lua
          resources
               config.yml
     moudule2
          manifest.yml
          scripts
               main.lua
          resources
               config.yml
     mouduleN
          manifest.yml
          scripts
               main.lua
          resources
               config.yml
modmanager命令改为mpm也就是模块包管理器
使用/mpm install modulepack.zip 就是安装modulepack.zip，也就是先进行格式校验，看看第一层是不是都是文件夹，有没有模块没有manifest.yml，如果格式不正确或有模块没有manifest.yml会直接返回模块包格式错误
mpm可以一次性安装多个包，比如/mpm install modulepack.zip xsdfg.zip
使用remove可以移除模块，移除模块可以使用模块名或模块的包名，如果两个模块重名了那显然必须使用包名，和install一样，remove也支持移除多个模块
使用uninstall可以卸载模块，卸载的模块会使用包名作为压缩包的名字，然后按照格式标准打包成zip，放到packages文件夹，然后移除被卸载的模块
uninstall也支持多个模块一起卸载，但是每个模块会单独打包，比如/mpm uninstall adc@dcc:1.0 dfg@adc:3.0 这个时候打包的压缩包是adc@dcc:1.0.zip和dfg@adc:3.0.zip，而不是一个压缩包
unpack和uninstall作用差不多，不过它会把多个模块打包在一起存放在压缩包里，使用格式是/mpm unpack <zipname> <modulename | packname>，多个模块就是/mpm unpack adc@dcc:1.0 dfg@adc:3.0
使用pack可以将多个或单个模块制作成模块包，和uninstall类似，但不会移除模块，比如使用/mpm pack adc@dcc:1.0 dfg@adc:3.0 这个时候打包的压缩包是adc@dcc:1.0.zip和dfg@adc:3.0.zip，而不是一个压缩包，而且对应的模块也不会被删除
使用packs可以把多个模块打包成一个包，使用/mpm packs <zipname> <modulename | packname>
，packs也不会删除模块
/mpm <enabled | disabled> <modulename | packname>保持不变，还是禁用启用模块
